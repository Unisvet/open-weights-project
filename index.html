💡 „Das Wissen teilen, nicht die Kontrolle abgeben.“
Willkommen im Cockpit Ihres KI-Workshops! Was sind Open-Weights-Modelle?

Künstliche Intelligenz beruht auf künstlichen neuronalen Netzen. Ein fertig trainiertes KI-Modell besteht im Kern aus einer mathematischen Struktur und Millionen oder Milliarden von Parametern – den sogenannten Modellgewichten (Weights). Diese Gewichte bestimmen, wie das Modell auf Eingaben reagiert.

Während bei Closed-Source-Modellen (z. B. ChatGPT, Claude) die Modellgewichte geheimgehalten werden und die Nutzung nur über eine Cloud-Schnittstelle (API) des Herstellers erfolgt, werden bei Open-Weights-Modellen (z. B. Googles Gemma-Familie) die gelernten Gewichte öffentlich zur Verfügung gestellt.

Der große Vorteil für KMU: Sie können diese Modelle herunterladen, lokal auf eigener Hardware ausführen und nach Belieben anpassen (Feintuning). Es fließen keine Firmendaten ins Internet ab.

Closed-Source APIs vs. Lokale Open-Weights Modelle

Closed-Source API

Cloud-Schnittstellen

z.B. GPT-4, Claude

  • Datenabfluss: Jede Anfrage verlässt das Unternehmen und wandert auf externe Server der US-Anbieter.
  • Laufende API-Gebühren: Abrechnung erfolgt nach Nutzung (Tokens), was schwer kalkulierbare Kosten erzeugt.
  • Hersteller-Abhängigkeit: Updates können das Modell-Verhalten unvorhersehbar ändern oder sperren.
Open-Weights

Lokale Bereitstellung

z.B. Gemma 4 12B Unified

  • 100% Datensouveränität: Kompletter Offline-Betrieb. Keine Betriebsgeheimnisse verlassen Ihr Haus.
  • Null API-Kosten: Einmalige Anschaffung der Hardware (z.B. moderner Arbeits-Laptop), danach unbegrenzte kostenfreie Nutzung.
  • Volle Kontrolle: Das Modell gehört Ihnen. Keine Zensur, kein plötzliches Abschalten, Möglichkeit des Feintunings.

Flexible Deployment-Umgebungen für Gemma 4

Ob maximale Datensouveränität vor Ort, skalierbare Cloud-Infrastruktur oder schnelle Browser-Experimente – Gemma 4 passt sich flexibel den Sicherheits- und Leistungsanforderungen Ihres KMU an.

1. Local & On-Device

Für absolute Privatsphäre, Offline-Fähigkeit und null API-Kosten laufen Gemma 4 Modelle direkt auf Ihrer eigenen Hardware.

  • Desktop & Laptop: Nutzen Sie quantisierte Gemma 4 Checkpoints (verfügbar in E2B, E4B, 12B, 26B & 31B) über extrem populäre und anwenderfreundliche Tools wie Ollama oder LM Studio.
  • Edge & Mobile: Die Modelle sind optimiert für den mobilen Betrieb auf iOS, Android (über das eingebaute Android AICore) und kompakte Edge-Devices wie den Raspberry Pi 5.
  • Native Apps: Mac-Anwender können über die Google AI Edge Gallery modernste Modelle hardwarebeschleunigt auf Apple-Silicon-GPUs (M1/M2/M3/M4) ausführen.

2. Cloud (Google Cloud)

Für unternehmensweiten Zugriff, Kunden-Anwendungen und zentrale Governance betreiben Sie Gemma 4 skalierbar in der Cloud.

  • Serverless Scale: Stellen Sie Gemma 4 über Cloud Run mit dedizierten GPU-Beschleunigern (wie NVIDIA L4) bereit – Sie zahlen sekundenpräzise nur bei aktiver Nutzung.
  • Enterprise Control: Nutzen Sie den Google Cloud Model Garden (Vertex AI / Gemini Enterprise Agent Platform), um eigene Ressourcen zu provisionieren und volle Kontrolle über die Infrastruktur zu behalten.
  • Custom Orchestration: Orchestrieren Sie komplexe High-Performance-Szenarien via Google Kubernetes Engine (GKE) mit vLLM oder TPUs.

3. Managed & Ecosystem

Für sofortige Tests ohne jeglichen Setup-Aufwand greifen Sie über fertige APIs und das globale KI-Ökosystem auf Gemma 4 zu.

  • Google AI Studio: Experimentieren Sie blitzschnell und kostenlos in einer Browser-Oberfläche direkt im Google AI Studio.
  • Zero-Setup APIs: Nutzen Sie kostengünstige, OpenAI-kompatible SaaS-Modellschnittstellen über Anbieter wie DeepInfra.
  • Cloud Partners & Fine-tuning: Greifen Sie auf Partner-Plattformen wie Amazon Bedrock oder Microsoft Azure AI Foundry zu, laden Sie Weights über Hugging Face und feintunen Sie hocheffizient mit Unsloth.

🦙 Wie arbeitet Ollama? (Hintergrund-Architektur)

Sie haben die absolut richtige Intuition! Wenn Sie Ollama nutzen, läuft alles vollständig lokal auf Ihrer eigenen Maschine. Es gibt keinen Cloud-Zwang, keine permanente Internetverbindung und keine Ihrer sensiblen Prompt-Daten verlässt Ihren Rechner.

📦 1. Was wird heruntergeladen? (Programm & Gewichte)

Das Programm (Ollama Engine)

Wenn Sie Ollama installieren, laden Sie ein leichtgewichtiges, hochoptimiertes und fertig kompiliertes Programm (geschrieben in Go und C++) auf Ihr System herunter.

  • Server-Client-Architektur: Enthält einen Hintergrund-Server (Daemon) und ein Kommandozeilen-Tool (CLI).
  • Das Herzstück: Unter der Haube nutzt Ollama llama.cpp – eine extrem effiziente C/C++-Bibliothek, die LLMs auf Consumer-Hardware mit maximaler Geschwindigkeit ausführt.
Die Modell-Gewichte (Model Weights)

Wenn Sie ein Modell mit ollama run ausführen, werden die mathematischen Gewichte (die "Intelligenz" des Modells) einmalig aus der offiziellen Ollama-Registry heruntergeladen.

  • Das GGUF-Format: Ein hochgradig optimiertes Format, das alle Komponenten (Gewichte, Tokenizer, Hyperparameter) in einer einzigen Datei bündelt.
  • Quantisierung (Kompression): Nutzt standardmäßig 4-Bit-Quantisierung. Ein Modell schrumpft z. B. von 30 GB auf 4-5 GB, ohne spürbare Qualitätseinbußen, und passt perfekt in den RAM.

🧠 2. Wie werden die Gewichte geladen? (RAM vs. VRAM)

Sobald Sie eine Anfrage stellen, liest Ollama die Modelldatei von Ihrer SSD und lädt sie in den Arbeitsspeicher. Dabei nutzt Ollama ein intelligentes Speicher-Management:

💾 Festplatte (SSD)
➔ Einmalig beim Start laden
⚡ Arbeitsspeicher (RAM / VRAM)
Szenario A: Voller VRAM-Modus

Wenn die GPU genügend Videospeicher (VRAM) besitzt, wird das Modell komplett dorthin geladen. Dies liefert die schnellste Performance.

Szenario B: Hybrid-Modus

Ist der VRAM knapp, teilt Ollama das Modell dynamisch auf. Ein Teil der neuronalen Schichten läuft auf der GPU, der Rest auf der CPU (RAM).

Szenario C: Reiner CPU-Modus

Ohne dedizierte GPU läuft das Modell vollständig im normalen System-RAM über den Hauptprozessor (CPU). Funktioniert dank llama.cpp dennoch flüssig.

⏱️ Ressourcenschonend: Wenn Sie Ollama eine Zeit lang (standardmäßig 5 Minuten) nicht benutzen, entlädt der Server das Modell automatisch wieder aus dem RAM/VRAM. Dadurch blockiert Ollama keine Systemressourcen, wenn Sie es nicht brauchen.

🔄 3. Wie fließen die Datenströme? (Der lokale Datenfluss)

Da alle Komponenten lokal agieren, bleibt der gesamte Datenfluss auf Ihr System beschränkt. Die Datenströme verlassen zu keinem Zeitpunkt Ihr lokales Netzwerk.

1
Eingabe (Prompt): Sie tippen eine Frage im Terminal ein oder senden sie über ein GUI (z. B. Open WebUI, Chatbox).
2
Lokaler API-Call (HTTP): Die UI sendet die Anfrage per lokalem Netzwerk-Protokoll (HTTP POST) an den Ollama-Server auf http://localhost:11434/api/chat.
3
Tokenisierung: Die Ollama-Engine übersetzt Ihre Wörter in Zahlen (Tokens), mit denen das neuronale Netz rechnen kann.
4
Verarbeitung (Inferenz): Die CPU und/oder GPU verarbeiten die Token Schicht für Schicht mithilfe der im RAM/VRAM liegenden Modellgewichte.
5
Detokenisierung: Die berechneten Wahrscheinlichkeiten werden von der Engine wieder in lesbare Wörter zurückübersetzt.
6
Ausgabe (Streaming): Ollama schickt die Antwort Token für Token in Echtzeit über die lokale Verbindung an die UI zurück.

Kriterium Lokaler Betrieb (Ollama) Cloud-Modelle (z.B. ChatGPT, Claude)
Ausführung Komplett auf Ihrer lokalen Hardware (CPU/GPU). Auf gigantischen Server-Clustern des Anbieters.
Ort der Gewichte Gespeichert auf Ihrer SSD, geladen in Ihren RAM. In den Rechenzentren des Anbieters.
Datenschutz 100 % privat. Keine Daten fließen ins Internet. Eingaben werden übertragen und ggf. gespeichert/verwendet.
Internet Nein. Nur einmalig für den Modell-Download benötigt. Ja. Permanente, stabile Verbindung zwingend notwendig.
Kosten Komplett kostenlos (außer Strom). Oft abobasiert oder nutzungsabhängig bezahlt (API-Kosten).
