Open-Weights-Modelle für KMU einfach erklärt

Willkommen beim interaktiven Begleit-Leitfaden für Ihren KI-Workshop. Erfahren Sie, wie kleine und mittlere Unternehmen (KMU) dank moderner Open-Weights-Modelle wie Gemma 4 12B Unified volle Datensouveränität behalten, komplett offline arbeiten und kostengünstig maßgeschneiderte KI-Lösungen implementieren können.

Was sind Open-Weights-Modelle?

Künstliche Intelligenz beruht auf künstlichen neuronalen Netzen. Ein fertig trainiertes KI-Modell besteht im Kern aus einer mathematischen Struktur und Millionen oder Milliarden von Parametern – den sogenannten Modellgewichten (Weights). Diese Gewichte bestimmen, wie das Modell auf Eingaben reagiert.

Während bei Closed-Source-Modellen (z. B. ChatGPT, Claude) die Modellgewichte geheimgehalten werden und die Nutzung nur über eine Cloud-Schnittstelle (API) des Herstellers erfolgt, werden bei Open-Weights-Modellen (z. B. Googles Gemma-Familie) die gelernten Gewichte öffentlich zur Verfügung gestellt.

Der große Vorteil für KMU: Sie können diese Modelle herunterladen, lokal auf eigener Hardware ausführen und nach Belieben anpassen (Feintuning). Es fließen keine Firmendaten ins Internet ab.

Interaktiver Gewichte-Visualisierer

Fahren Sie mit der Maus über die Knoten (Neuronen) oder verändern Sie die Schieberegler, um zu verstehen, wie die Modellgewichte die Signalübertragung im Netz steuern. In einem echten Modell wie Gemma 4 12B Unified gibt es 12 Milliarden solcher Verbindungen!

Gewichtssteuerung

Durch Erhöhung des Gewichts einer Verbindung wird das Signal zwischen den Neuronen verstärkt. Bei der Texterstellung wird so bestimmt, welches Wort als nächstes mit der höchsten Wahrscheinlichkeit folgt.

Closed-Source APIs vs. Lokale Open-Weights Modelle

Closed-Source API

Cloud-Schnittstellen

z.B. GPT-4, Claude

  • Datenabfluss: Jede Anfrage verlässt das Unternehmen und wandert auf externe Server der US-Anbieter.
  • Laufende API-Gebühren: Abrechnung erfolgt nach Nutzung (Tokens), was schwer kalkulierbare Kosten erzeugt.
  • Hersteller-Abhängigkeit: Updates können das Modell-Verhalten unvorhersehbar ändern oder sperren.
Open-Weights

Lokale Bereitstellung

z.B. Gemma 4 12B Unified

  • 100% Datensouveränität: Kompletter Offline-Betrieb. Keine Betriebsgeheimnisse verlassen Ihr Haus.
  • Null API-Kosten: Einmalige Anschaffung der Hardware (z.B. moderner Arbeits-Laptop), danach unbegrenzte kostenfreie Nutzung.
  • Volle Kontrolle: Das Modell gehört Ihnen. Keine Zensur, kein plötzliches Abschalten, Möglichkeit des Feintunings.

Lokale Voraussetzungen einrichten

Um Gemma 4 12B Unified offline auf Ihrer eigenen Hardware auszuführen, nutzen wir zwei extrem performante, moderne Tools: Ollama (für die Ausführung der KI) und uv (für die schnelle Ausführung des Python-Codes).

1. Ollama installieren & Modell laden

Ollama ist eine leichtgewichtige Engine, die LLMs lokal ausführt. Laden Sie sie von ollama.com herunter und installieren Sie sie. Öffnen Sie danach Ihr Terminal (PowerShell unter Windows) und laden Sie das Modell Gemma 4 herunter:

PowerShell / Terminal 
# Gemma 4 12B Unified Modell herunterladen (ca. 8 GB)
ollama pull gemma4:12b

# Optional: Nomic Embed Text für die lokale Vektorerstellung laden (ca. 270 MB)
ollama pull nomic-embed-text

2. uv (Moderner Python Package Manager)

uv ist ein ultraschneller, moderner Paketmanager von Astral. Er ermöglicht es uns, Python-Skripte inklusive ihrer Abhängigkeiten absolut zuverlässig und ohne manuelle Umgebungseinrichtung auszuführen.

Windows Installation:

PowerShell (Administrator) 
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Das sichere Offline-Firmenwiki für den Maschinenbau

Das Problem: Ein mittelständischer Industriebetrieb verfügt über jahrzehntelanges Wissen: tausende PDFs voller ISO-Normen, Wartungsprotokolle, Betriebsanleitungen und alte Baupläne. Neues Personal verbringt oft Stunden mit der Suche nach spezifischen Details.

Die Lösung

Ein lokales RAG-System (Retrieval-Augmented Generation) verbindet das Offline-Modell Gemma 4 12B Unified mit dem Dokumentenordner des Betriebs. Ein Mitarbeiter gibt ein:
„Welche Toleranzmaße galten bei der Ventil-Baugruppe X aus dem Projekt von 2026?“

Der Nutzen

Innerhalb von Sekunden liefert die KI die exakte Antwort inklusive Quellenangabe (z.B. Dateiname & Seite). Da alles lokal auf dem Server läuft, verlässt kein einziges Betriebsgeheimnis das Werksgelände.

Praktischer Python-Code mit uv (Single File Script)

Dank des PEP 723 Standards von uv können wir alle Abhängigkeiten direkt im Kopf des Skripts deklarieren. Sie müssen nichts manuell installieren! Erstellen Sie eine Datei namens local_rag.py und fügen Sie folgenden Code ein:

local_rag.py (Python Script) 
# /// script
# requires-python = ">=3.11"
# dependencies = [
#   "langchain-ollama>=0.2.0",
#   "langchain-chroma>=0.1.2",
#   "langchain>=0.2.0",
#   "pypdf>=4.2.0",
# ]
# ///

import os
from langchain_community.document_loaders import PyPDFDirectoryLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_chroma import Chroma
from langchain_ollama import OllamaEmbeddings, ChatOllama
from langchain.chains import create_retrieval_chain
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain_core.prompts import ChatPromptTemplate

# 1. PDFs aus lokalem Ordner laden (Erstellen Sie einen Ordner 'dokumente/')
os.makedirs("dokumente", exist_ok=True)
print("Bitte legen Sie PDFs in den Ordner 'dokumente'...")
loader = PyPDFDirectoryLoader("dokumente/")
docs = loader.load()

if not docs:
    print("Keine PDFs gefunden. Initialisiere RAG mit Demo-Wissen...")
    from langchain_core.documents import Document
    docs = [
        Document(page_content="ISO_Norm_Ventil_X_2026: Die zulässigen Toleranzmaße bei der Ventil-Baugruppe X betragen für das Gehäuse maximal +0.02 mm und für den Ventilteller minimal -0.01 mm.", metadata={"source": "ISO_Norm_Ventil_X_2026.pdf"}),
        Document(page_content="Wartungsprotokoll_Pumpe_Y_2025: Die Vakuumpumpe Typ Y ist alle 250 Betriebsstunden zu schmieren und nach 2000 Betriebsstunden einer vollständigen Revision zu unterziehen.", metadata={"source": "Wartung_Vakuumpumpe_2025.pdf"})
    ]

# 2. Text in mundgerechte Stücke (Chunks) zerteilen
text_splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=100)
chunks = text_splitter.split_documents(docs)

# 3. Lokale Embeddings generieren und in Chroma Vektordatenbank speichern
print("Generiere Offline-Embeddings via Ollama (nomic-embed-text)...")
embeddings = OllamaEmbeddings(model="nomic-embed-text")
vector_store = Chroma.from_documents(chunks, embeddings, persist_directory="./chroma_db")

# 4. Prompt Template für Gemma 4 erstellen
prompt_template = ChatPromptTemplate.from_template("""
Du bist ein hochpräziser Assistenz-Algorithmus für den Maschinenbau. 
Nutze ausschließlich den beigefügten Kontext, um die Frage des Mitarbeiters sachlich zu beantworten.
Wenn die Antwort nicht im Kontext zu finden ist, sage ehrlich, dass das Wissen fehlt.

Kontext:
{context}

Frage: {input}

Detaillierte Antwort mit genauer Quellenangabe:""")

# 5. Gemma 4 offline über Ollama ansprechen
print("Lade Gemma 4 12B Unified offline...")
llm = ChatOllama(model="gemma4:12b", temperature=0.2)

# 6. RAG-Pipeline verbinden & abfragen
retriever = vector_store.as_retriever(search_kwargs={"k": 2})
document_chain = create_stuff_documents_chain(llm, prompt_template)
retrieval_chain = create_retrieval_chain(retriever, document_chain)

query = "Welche Toleranzmaße galten bei der Ventil-Baugruppe X aus dem Projekt von 2026?"
print(f"\nFrage an lokales RAG: '{query}'")
print("KI sucht in lokalen PDFs und antwortet offline...")

response = retrieval_chain.invoke({"input": query})
print("\n--- OFFLINE KI ANTWORT ---")
print(response["answer"])

So führen Sie dieses Skript blitzschnell aus:

PowerShell 
uv run local_rag.py
